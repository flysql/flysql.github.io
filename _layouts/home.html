Einfachste Integration

von digitalen Dienstleistungs Systemen, Avaloq Abfragen oder Avaloq Workflows in einen Testflow

  • E-Banking - Ausführen eines Zahlungsauftrags oder Ausführen eines Wertschriftenorders in Avaloq
  • E-Banking - Prüfen ob die im E-Banking erfassten Daten auf dem Order im Avaloq abgelegt wurden
  • Skalierbare Tests um das Kostenregelwerk im Avaloq zu testen

Jeder Experte kann unabhängig

in seinem jeweiligem Fachgebiet, zum Erfolg beitragen

  • Avaloq und SQL Skripte werden durch Certified Avaloq Professional bereitgestellt
  • Testautomation Entwickler binden FlySql mit dem Tool ihrer Wahl (Tosca, Java/Junit, Python, ...) ein
  • Test Experten haben bei der Testausführung transparent Zugriff auf die Testschritte mit FlySql
  • Avaloq Release Manager kontrollieren den FlySql Service - K Passwort, DB ZUgriffe, Skalierung

Volle Audit Sicherheit dank FlySql

  • SQL und Avaloq Scripte mit denen FlySql arbeitet, sind in einem Git Repository abgelegt. Änderungen sind transparent und können dem 4 Augen Prinzip unterworfen werden
  • Abfragen werden durch FlySql protokolliert
  • Systempasswörter (zum Beispiel das K Passwort) sind nur dem FlySql Service bekannt und somit geschützt

Volle Skalierung

der Testautomatisierung dank FlySq

  • Einfachste Einbindung der verschiedensten Fachexperten, jeder arbeitet in seinem Bereich unabhängig am Gesamtkonzept
  • 100% Verwendung von Industrie Standardlösungen (Git, REST, Docker, Java)
  • Keine Installation auf den Datenbank Systemen notwendig
  • Beliebige Skalierung der FlySql Services mittels Docker

Dynamische Testdaten

und Zugriffe auf das Kernbankensystem dank FlySql

  • Abfrage und Generierung von Testdaten und Systemzuständen, um dynamischen Daten / Zustände abzufragen oder zu generieren
  • Ausführen dieser SQL Typen in einem Testflow, alle SQL Typen parametrisierbar falls notwendig
    • SQL Abfragen, zum Beispiel select isin from obj_asset ... where ${EBANKINGID} = 'fly-sql-ebanking')
    • PL/SQL Statements
    • Avaloq Statements, zum Beispiel lookup.asset('${ISIN}').name)
    • Avaloq Script, zum Beispiel with new mem_doc_xfermon ... tdoc := ${AMOUNT} ... return doc.id)
  • Beispiel E-Banking Testfall
    • Abfrage des Kontostandes oder des aktuellen Preises eines Wertpapiers
    • Simulieren einer Einzahlung um einen minimalen Kontostand für den Testfall sicherzustellen
    • Ausführen eine via E-Banking erfassten Börsenauftrags im Kernbankensystem
